vex-bridge for Revit

Push the model you have open to architur with one click. Architur is semantic version control for BIM — like GitHub, but the diff knows the difference between a moved wall and a renumbered ID.

One-time setup

1. Install Vex Atlas. The early access installer and bundle include the Revit add-in, the vex CLI, and the local vex-bridge daemon together.
2. In Revit, open the Add-Ins tab and click Pair this device on the vex-bridge panel. Sign in or create an account, then approve the device on studio.planmorph.software when prompted.
3. For the full early access walkthrough, open EarlyAccessInstall.html from this same folder.

Pushing a model

1. Open the model you want to push.
2. Click Push to architur.
3. Paste your project ID (find it on studio.planmorph.software under Project ▸ Settings) and click Push.

Privacy

The plug-in talks only to http://127.0.0.1:7878 — your local vex-bridge daemon. The daemon talks to architur over SSH using a key generated on this machine that never touches our servers in plaintext. We do not collect telemetry from inside Revit.
